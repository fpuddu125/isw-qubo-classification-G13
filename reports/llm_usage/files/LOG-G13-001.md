```text
The error you are seeing indicates that the column `policy_code` still produces NaN values after normalization.  
This happens when a feature has zero variance (all values are identical), because the standard deviation becomes zero and the normalization step produces divisions by zero, resulting in NaN.

To fix this, you need to correctly detect constant columns during the mean/std computation.  
Right now your code uses:

std = math.sqrt(var) if var > 0 else 1.0

This forces the standard deviation to 1.0 even when the variance is actually zero, which prevents the preprocessing step from identifying constant columns. As a result, the column is kept and later produces NaN values.

You should replace that line with the following logic:

if var == 0:
    std = 0.0   # mark constant column
else:
    std = math.sqrt(var)

This way, columns with zero variance will correctly have std = 0.0, and your preprocessing script will be able to remove them before normalization.

After applying this change, re-run the preprocessing to regenerate the train and test files, and then run the automatic tests again. The normalization test should pass.
```
